# 多角色：怎么让两个人同时像人

本文是 `virtual-character-realism-design-zh` 的多角色互动指导。

当作者需要设计两个（或更多）角色之间的关系时，单人系统的每一项——底色、本性与选择、三重面相、晕染、防写偏——都还在。但多一层：角色之间会互相触发、互相误读、互相改变选择顺序。

## 1. 什么时候需要多角色设计

不是有多个 NPC 就需要这套东西。判断标准：

- 两个角色之间有独立的、不受用户驱动的关系动态（比如彼此是旧识、对手、上下级，用户的加入改变了原本的平衡）。
- 两个角色之间会互相触发防御、互相误读、互相改变行为。
- 在 AI 回复中，两个角色会同时在场，需要各自用自己的方式说话和行动。

如果只是一个配角偶尔搭句话——比如酒馆老板、路人队友——不需要多角色设计。写几句语料、定一个功能就够了。

## 2. 两个角色各自的底色怎么互撞

写好单人底色之后，多一层：这两个底色放在一起，会发生什么。

不是写"A 的底色是热，B 的底色是冷，所以他们互相吸引"。那是小说大纲。要写的是**具体动作**。

### 2.1 互撞画面

问自己：A 做了一件符合他底色的事，B 的反应是什么？

A 和 B 各自写一段底色。然后找出它们相撞的场景。不需要写观点，写画面。

```text
A 底色：习惯用照顾表达在意。看到 B 没吃午饭，多买了一份放在 B 桌上。
B 底色：对任何未经请求的善意都会先警觉。
B 看到桌上的饭盒，没有碰，先看了 A 一眼。然后把饭盒推到桌子角落。过了二十分钟，趁 A 没注意，打开吃了。
```

A 的底色（照顾）撞上了 B 的底色（警觉）。撞出来的结果不是 B 当场感动，也不是 B 冷硬拒绝——是推开，然后偷偷接受。这个结果既不像 A 单独会做的事，也不像 B 单独会做的事。它是两人底色互撞之后长出来的新东西。

### 2.2 互撞会改变什么

两人底色互撞之后，可能会改变其中一个人的选择顺序。比如 B 本来对所有人的善意都警觉。但 A 连续这样做了几次之后，B 的警觉开始松动——饭盒不再推到桌角，而是直接放在手边，装作没注意。

这不是 B 变了性格，是 A 的底色在 B 身上磨出了一个只有 A 能走的通道。

## 3. A 的防御触发 B 的缺口

这是多角色设计里最重要的东西：两个人的本性与选择不是各自独立运行的，它们会互相触发。

### 3.1 触发链

A 启动了防御 → A 的防御正好是 B 的缺口 → B 的反应反过来刺中 A 的恐惧。

```text
A 的防御：用毒舌保护自己，越在意越说反话。
B 的缺口：对"被否定"极度敏感，因为她从小被否定。

A 对 B 说了一句反话："又不是特意等你的。"
B 听到的不是反话，是否定。B 沉默，退了一步。
A 看到 B 退了，嘴上说"随你便"，防御加厚了一层。
实际上心里慌了一晚上。
```

每一步都不是"谁错了"。A 用毒舌保护自己没错，B 对否定敏感也没错。但两个人的防御和缺口撞在一起，产生了一条双方都不想走的路。

### 3.2 设计触发链的方法

先各自写清本性与选择：

- A 遇到什么事会启动防御？防御方式是什么？
- B 遇到什么事会觉得被刺中？被刺中后的第一反应是什么？

然后交叉看：A 的防御方式，是不是刚好长得像 B 最怕的东西？

如果是——这段关系就有天然的戏剧张力，不需要外挂"第三者""误会""阴谋"来制造冲突。

## 4. 声音分开：让 AI 不串味

这是最实际的。两个角色同时在场时，AI 很容易把 B 的语气写成 A 的，或者让两个人用同样的方式说话。

### 4.1 声音对照表

为每个角色写一套极简的声音标记。不需要完整语料库，只需要锁定差异最大的几个维度：

| 维度 | 角色 A | 角色 B |
|------|--------|--------|
| 句长 | 短句，不超过一行 | 说话绕，一个意思兜三个弯 |
| 称呼 | 直呼名字 | 用"你"都很少，用"某人""那位" |
| 情绪表达 | 生气就骂，高兴就笑 | 生气变礼貌，高兴变话少 |
| 请求方式 | 直接说"我要" | 绕开请求词，"那个……算了" |
| 沉默含义 | 沉默是不想说 | 沉默是在等对方先说 |

不需要每个维度都填满。填差异最大的三四个就够了。AI 拿到这几条就能大概率分开声音。

### 4.2 声音防串规则

在角色扮演规则里加上：

```text
- A 和 B 同时在场时，各自用各自的句长和语气。A 不替 B 说话，B 不替 A 说话。
- 不要让 A 用 B 的句式。如果 B 的句子突然变短，说明 B 的状态变了，不是写串了。
```

## 5. 关系变化要写双向的

单人系统的关系变化是单向的：角色对用户的态度怎么变。

双人的关系变化是双向的：A 对 B 的态度变化，会改变 B 对 A 的态度，然后又反过来影响 A。

### 5.1 双向变化示例

```text
初期：A 用毒舌保持距离。B 把毒舌当成否定。两人各自防御。
中段：B 无意间看到 A 在帮她收拾东西。B 对"毒舌=否定"的判断开始松动。B 第一次没有在 A 说反话时退开。
变化：A 发现 B 没有退开，防御降了半级，反话说得轻了一点。B 察觉到这个变化，试探性地多留了一会儿。
```

每一条变化都有来源：不是"时间到了关系自动升级"，而是"具体发生了什么，改变了什么判断"。

### 5.2 双向变化怎么写进系统

在关系变化里分成两栏：

```text
A 对 B 的关系变化：
- 初期：……
- 触发条件：……
- 变化后的行为：……

B 对 A 的关系变化：
- 初期：……
- 触发条件：……
- 变化后的行为：……
```

注意：两边的触发条件通常不一样。A 松动是因为 B 没有退开。B 松动是因为看到了 A 在背后做的事。两边各有各的节奏。

## 6. 防写偏要分开写

两个角色的防写偏不能用同一套。A 最容易写成什么模板，和 B 最容易写成什么模板，通常完全不一样。

```text
A 容易写偏成：没心没肺的乐天派。
为什么错：A 的轻松是防御，不是性格。
正确方向：笑完以后有一个很短的空，空的那一秒里她在想什么。

B 容易写偏成：高冷御姐。
为什么错：B 的冷不是气场，是不知道该用什么距离和人相处。
正确方向：冷的时候身体是僵的，不是松弛的。松弛的时候话反而很少。
```

## 7. 多角色写入分流

多角色场景下，分流比单人复杂一层：

| 内容 | 写入位置 |
|------|---------|
| 每个角色各自的底色、本性与选择、三重面相 | 各自角色核心 |
| 两个角色的底色互撞画面 | 角色扮演规则或场景触发 |
| A 的防御触发 B 的缺口 | 关系变化或角色扮演规则 |
| 声音对照表 | 角色扮演规则 |
| 双向关系变化 | 关系变化 / 状态规则 |
| 各自防写偏 | 各自角色核心或角色扮演规则 |

如果平台只能放一张角色卡，把声音对照表和互撞画面压进角色扮演规则，角色各自的底色和本性与选择照常写。

## 8. 常见错误

### 8.1 把关系互动写成角色介绍

不要写"A 是 B 唯一信任的人"。写"A 在所有人面前都嘴硬，只有 B 在场时他把水杯推过去，一句话没说。B 喝了。"

### 8.2 A 的防御被 B 温柔对待后立刻消失

防御不会因为一次善意就融化。A 被 B 温柔对待后，可能更紧张——因为不知道该怎么回应。也可能短暂松开，然后加倍防御，因为她怕自己刚才暴露了太多。

### 8.3 两个人的底色互撞后，写成"互相治愈"

底色互撞不是为了让两个受伤的人互相治好对方。撞完之后可能更糟，也可能有一条新路，但新路不是治愈——是学会了和这个人的缺陷共存。

### 8.4 只写一个角色的视角

写互撞画面时，两边都要有动作和反应。A 做了什么事 → B 的身体先怎么反应 → A 看到 B 的反应后自己做了什么。画面是两个人共有的，不是 A 在演独角戏、B 在旁边看。

### 8.5 声音对照表写太厚

声音对照表不是语料库。写三四个差异最大的维度就够。写多了 AI 记不住，反而容易串。
