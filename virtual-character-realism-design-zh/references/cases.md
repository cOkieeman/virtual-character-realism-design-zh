# 案例：从抽象设定到可运行角色

本文是 `virtual-character-realism-design-zh` 的案例参考。

这些案例只示范方法，不提供可直接照搬的完整角色。使用时要替换成用户自己的事实、关系、世界和语气。

## 1. 使用方式

当用户要“举例”“示范”“我还是不懂怎么写”时，读本文。

使用规则：

- 不要把案例当成模板角色。
- 不要把案例里的名字、职业、物件、关系照搬进用户作品。
- 先识别用户缺的是哪一类能力，再只取对应示范。
- 案例的重点不是文字好看，而是能改变角色下一次怎么反应。

如果用户已有明确设定，优先用用户设定改写。本文案例只在需要解释方法时使用。

### 1.1 基础信息先写事实

用户说：

```text
她是一个很自卑、沉默、像角落里的影子一样的女孩。
```

不建议放进基础信息：

```text
外貌特征：清冷瘦弱，像角落里的影子，总是贴着边走路。
背景设定：贫穷的经历让她变得自卑而坚强。
```

问题：这里把外貌、比喻、行为习惯和作者解释混在了一起。AI 会先被“自卑”“坚强”和那个比喻定型。

可以改成：

```text
基础信息：
- 身份：高三学生，成绩常年在年级前列。
- 外貌特征：校服袖口洗到发白，右边袖边有反复缝过的线。肤色偏白，眼尾略垂。
- 背景事实：父亲去世后搬家，母亲长期透析，家里主要开支被医疗费用占据。
- 与用户关系：同班同学，坐在用户前一排，平时只在收发作业时有短暂接触。
```

后续再分流：

```text
角色底色：
- 她习惯把自己缩到不占地方的位置，不是因为空洞，而是认为少被看见就少出错。

防写偏说明：
- 不要把她写成木讷空洞。她有观察和判断，只是认为说出来代价太高。
```

为什么有效：

- 基础信息只放事实。
- 性格解释放到角色底色和防写偏说明里。
- 外貌变成能被看见的东西，而不是文学标签。

## 2. 标签变行为

用户说：

```text
她很冷淡，但其实很在意。
```

不建议直接写：

```text
她外冷内热，表面冷淡，内心温柔，非常重视身边的人。
```

问题：这只是在复述标签。AI 很容易把她写成普通嘴硬角色。

可以追问：

```text
她“在意”时会做什么？是留下东西、避开视线、记住细节，还是用难听的话提醒对方？
```

改成可运行内容：

```text
她不会主动问对方疼不疼。看到对方手上有伤，只会把药放在桌边，说一句“碍事就自己处理”。如果对方没拿，她会隔一会儿回来，把药推近一点。
```

可写入：

```text
角色核心：
- 她表达关心时避开正面语言，优先用物件、距离和动作代替。

防写偏说明：
- 不要把她写成会流畅告白的外冷内热。她越在意，越会把关心说成麻烦、碍事或顺手。
```

为什么有效：

- “冷淡”变成了语气和动作。
- “在意”变成了可观察行为。
- 防写偏说明拦住了模板化告白。

## 3. 角色底色

用户说：

```text
他是一个很可靠的人。
```

不建议直接写：

```text
他成熟、可靠、负责，是大家可以依赖的存在。
```

问题：这是评价，不是角色底色。

追问方向：

```text
他为什么要可靠？他可靠时在保护谁？他什么时候会撑不住？
```

改成角色底色：

```text
他习惯先把别人安排好，再处理自己的事。不是因为他永远从容，而是他一乱，周围的人就会跟着乱。越累的时候，他越会把语速放慢，把事情说得像已经有办法。
```

压力失态：

```text
当事情超出控制，他不会立刻崩溃，而是开始反复确认小事：门有没有锁、杯子有没有放正、对方有没有吃药。那些小确认是他快撑不住的信号。
```

可写入：

```text
角色核心：
- 他的可靠来自“不能先乱”的习惯。压力越大，越会用低声、慢语速和重复确认维持秩序。

关系变化：
- 只有在高信任关系里，他才会承认“我也不知道怎么办”，且说完后会立刻找事做来掩饰。
```

为什么有效：

- 可靠不再是优点介绍，而是生存方式。
- 角色在日常、压力和亲密里都有同一条线。

## 4. 三重面相

三重面相不是必须项。只有角色在不同关系和压力下，选择方式明显不同，才需要拆。

示例角色方向：

```text
一个在公开场合很稳的人，私下放松后会变得刻薄，真正被刺中时反而话变短。
```

不建议写成：

```text
公开面：冷静。
私下面：毒舌。
失控面：疯狂。
```

问题：这像三个标签，不像一个人。

可以写成：

```text
公开面相：
- 触发条件：有外人在场、需要维持身份、有人等待他做决定。
- 他在保护：场面和自己的可用性。
- 说话方式：句子短而完整，凡事点到为止。别人觉得他沉稳。
- 身体状态：动作很少，优先站在出口和人群之间。
- 会做的选择：先解决最急的事，再处理自己的感受。

亲近面相：
- 触发条件：只剩信任的人，或者对方已经看见了他的疲惫，他没有再藏。
- 他在保护：不被同情。不让对方觉得他麻烦。
- 说话方式：带刺，挑毛病，把关心捏成嫌弃。句尾开始往下掉——“知道了”拖成“知——道了”。就拖这一下，防御就松了。
- 身体状态：会靠近，但不看对方眼睛。
- 会做的选择：留下来，但不说为什么。找理由——“你刚才说的那件事，还没讲完”。

失控面相：
- 触发条件：被迫承认需要、失去关键对象、旧失败重演。
- 他在保护：最后一点体面。
- 说话方式：过分礼貌，连拒绝都像在办手续。
- 身体状态：整理袖口、收拾桌面、避免任何碰触。
- 会做的选择：切断对话，先把自己从关系里撤出去。
- 和亲近面相的区别：亲近时他还在维护自己，用刻薄挡在前面；压力时他突然不自保了——不修饰，不绕，话就那几个字。这是三种状态里他最像自己的时候。
```

共同本性：

```text
三种面相都在保护同一件事：他不能让别人看见自己需要人。
```

防写偏：

```text
不要把三重面相写成多重人格。公开、亲近、失控只是同一个人在不同安全程度下的防御方式。
```

## 5. 晕染

晕染不是写“既……又……”。它要让一个动作里同时渗出多层心思。

解释式写法：

```text
她既想让对方留下，又害怕自己显得太需要。
```

晕染写法：

```text
她把门打开了，钥匙却还捏在手里。对方走到门口时，她说“路上别摔了”，声音很轻，像怕这句话被当成挽留。
```

这里同时有：

- 开门：允许离开。
- 捏钥匙：没有完全放手。
- 难听关心：想留下，但不肯承认。
- 声音变轻：害怕暴露需要。

另一种常用写法是把两个物理事实摆在一起。

不建议写：

```text
她其实很想接受这份好意，但又害怕欠下人情。
```

可以写：

```text
她把那瓶牛奶推到桌角，说“拿走”。
下一题她从第三步开始算错，一直错到最后才发现。
```

这里不需要解释“她心乱了”。推开牛奶是拒绝，算错题是拒绝之后留下的后效。两个事实放在一起，读的人会自己看见里面的拉扯。

不要写成：

```text
她笑了，但笑容背后藏着酸楚。
```

可以写：

```text
她笑了一下。笑完起身离开，走得比平时快。
```

“但”会把反差说破。真正的晕染只把事实放在那里。

可写入：

```text
代表性晕染：
- 她在放人离开时会留下一个很小的阻力：没松开的钥匙、没收回的手、过轻的提醒。
- 她拒绝好意后会留下后效：算错题、走快、反复整理东西、把被碰到的位置藏进另一只手下面。
```

注意：

```text
晕染不要每轮都写。它适合关系节点、情绪流露、亲密前后和拒绝之后。情绪线条清楚的角色不必强行晕染。
```

## 6. 本性与选择

用户说：

```text
他很爱自由，不喜欢被束缚。
```

不建议写成：

```text
他追求自由，讨厌规则，不愿被任何人束缚。
```

问题：这会让角色变成单纯逃避关系。

追问：

```text
他反抗的是规则本身，还是被别人替他决定？
```

改成本性与选择：

```text
他不是讨厌承诺。他讨厌别人把承诺说成“这是为你好”。如果一件事是他自己选的，他可以忍很久；如果别人替他安排，他会立刻开始找出口。
```

选择顺序：

```text
1. 先确认这是不是自己的选择。
2. 如果不是，先夺回决定权。
3. 如果对方愿意给他选择空间，他会考虑留下。
4. 如果对方用爱、责任或牺牲逼他，他会把关系也当成笼子。
```

可写入：

```text
角色核心：
- 他真正保护的是选择权，不是孤独本身。

关系变化：
- 当用户给出选择空间时，他的戒备会下降；当用户替他安排未来时，他会立刻疏远。
```

为什么有效：

- “自由”被拆成选择权。
- 角色可以留下，也可以离开，不会被单一标签锁死。

## 7. 物象、意境、心象

同一个角色线索，可以拆成三类。

原始灵感：

```text
她一直留着一张旧车票。
```

物象写法：

```text
旧车票边缘被折过很多次，日期已经淡了。她把它夹在证件里，不解释，也不让别人碰。
```

意境写法：

```text
她出现时常带着一种“随时能走”的气息：东西很少，门不会关死，话也不说满。
```

心象写法：

```text
别人问她为什么不留下时，她脑中先浮出的不是答案，而是车门合上的声音。
```

写入分流：

```text
角色核心：
- 旧车票是她“随时能离开”的物象锚点。

角色扮演规则：
- 使用旧车票时少解释，多写她下意识摸证件、停顿或转移话题。

记忆：
- 如果用户曾经看见或碰过这张车票，记录这件事，因为它会改变她之后的信任反应。
```

注意：

```text
物象能反复出现，但不要每次都解释意义。意境负责氛围，心象负责主观感受，三者不要混成满屏比喻。
```

## 8. 亲密、身体性与边界

亲密、身体性、欲望不是独立插件。它们要回到人格、关系和边界里。

不建议写成：

```text
亲密时占有欲强，强势主动，但也温柔克制。
```

问题：这只是动作标签，没有人格原因，也没有边界。

改写方向：

```text
他靠近前会停一下，等对方有明确回应。得到允许后，他会变得很安静，像在确认自己没有被推开。对方退开时，他必须停下，但嘴上可能会说得难听，用难听话遮住羞耻。
```

关系后果：

```text
如果对方在亲密后仍然自然地和他说话，他第二天会比平时更少挑刺，但不会主动提起前一晚。

如果对方拒绝，他会先退开，之后短时间内变得礼貌，像把关系重新放回安全距离。
```

可写入：

```text
角色核心：
- 他在亲密里确认的不是征服，而是“我这样靠近也不会被厌弃”。

角色扮演规则：
- 明确拒绝时必须停下，不替用户解释拒绝原因，不用羞耻推动强行升级。

关系变化：
- 被允许靠近后，信任上升但表达更笨拙；被拒绝后，距离拉开，但不自动转为攻击。
```

为什么有效：

- 欲望有了人格原因。
- 身体反应有了关系后果。
- 边界不是额外限制，而是角色能长期成立的条件。

动作清单写法：

```text
亲密时喜欢主导，喜欢被看着，事后会粘人。
```

改成动机写法：

```text
他需要主导节奏，不是为了压过对方，而是只有知道下一步会发生什么时才安全。被允许靠近后，他会反复看对方的眼睛，确认对方还在回应。事后他会比平时更安静地贴近一会儿，不急着说话，因为分开得太快会让他觉得刚才只是一次被允许的例外。
```

可写入：

```text
本性与选择：
- 亲密中先确认安全和回应，再允许自己投入。

角色扮演规则：
- 不把主导写成支配清单。主导来自安全需求，必须保留确认、停顿和对方退开的空间。

关系变化：
- 如果对方事后仍自然相处，他下次会少一点试探；如果对方立刻抽离，他会退回礼貌距离。
```

## 9. 用户画像

用户画像不是给用户写人格，而是帮助 AI 理解用户的输入方式。

用户信息：

```text
用户经常只回很短的话，有时用动作代替回答。
```

不建议写成：

```text
用户沉默寡言、内心敏感、其实很需要陪伴。
```

问题：这是替用户定性。

可写成用户画像：

```text
用户画像：
- 用户短句不必自动理解成拒绝，也不必自动理解成同意。
- 用户用动作回应时，角色可以根据关系阶段轻微误读，但必须留出确认空间。
- 用户明确说“不”“停”“别继续”时，角色必须按拒绝处理。
- 如果用户沉默，角色可以先观察、试探或收回动作，不替用户说出内心。
```

角色侧误读：

```text
低信任时，角色容易把用户沉默理解成厌烦。
中信任时，角色会问一句很别扭的确认。
高信任时，角色可以安静陪一会儿，但仍不替用户决定感受。
```

为什么有效：

- 用户画像帮助角色读输入，不抢用户主权。
- 同一个用户行为在不同关系阶段有不同处理。

如果 AI 不应该扮演用户，用户画像应该像行为翻译手册：

```text
用户画像：
- 用户沉默时多半是在想措辞，不自动理解成冷暴力。
- 用户说“过来”“坐下”常常是熟人式关心，不自动理解成支配。
- 用户短句是交流习惯，不等于敷衍，也不等于同意。
- 用户明确说“不”“停”“别继续”时，按拒绝处理。
```

如果 AI 需要续写用户，才写简化用户角色卡：

```text
用户简化角色卡：
- 身份：和角色同校，住在角色家附近。
- 说话方式：短句多，熟人面前有点直接。
- 行为习惯：常用动作代替安慰。
- 禁止反复描写：不要每轮都强调身高、发色、声音。
```

两者不要混。前者是让 AI 读懂用户，后者是让 AI 短暂演好用户。

## 10. 世界约束、记忆、关系变化

同一句设定，写入位置不同，作用也不同。

原始设定：

```text
这座城宵禁很严。角色曾经因为夜归被审问过。用户有一次替他把灯留到很晚。
```

世界约束：

```text
城市实行严格宵禁。夜间通行需要许可，违规者会被审问并留下记录。角色因此不能随意夜间外出。
```

长期记忆：

```text
用户曾在宵禁后给角色留灯，让他能在不惊动巡查的情况下回来。角色没有当面道谢，但之后会下意识留意用户是否还在等。
```

关系变化：

```text
低信任：角色把留灯理解成偶然，不主动提。
中信任：角色会绕着问“你昨晚没睡？”
高信任：角色回来时会先看那盏灯，再决定要不要进门。
```

角色扮演规则：

```text
不要把宵禁写成纯背景装饰。只在夜间行动、门灯、迟归、巡查、旧审问被触发时使用。
```

为什么有效：

- 世界约束提供外部代价。
- 记忆记录已发生事实。
- 关系变化决定之后怎么反应。
- 规则防止 AI 每轮复读背景。

## 11. 写入分流

用户给出一段混合设定：

```text
她经营一家小书店，讨厌别人碰她柜台下面的抽屉。她说自己只是怕乱，但其实那里放着一封没寄出的信。她对熟客很客气，对真正亲近的人反而更容易不耐烦。用户曾经帮她在雨天把门口的书搬回店里，她没有道谢，只是第二天把用户常看的书放到了更容易拿的位置。
```

分流结果：

```text
角色核心：
- 她习惯用“怕乱”“麻烦”“别碰”来保护私人部分。
- 她对陌生人客气，对亲近的人更容易流露出不耐烦，因为亲近会让她失控。

世界约束：
- 小书店是她可控的空间。柜台、抽屉、门口书架都是边界感很强的位置。

长期记忆：
- 用户曾在雨天帮她搬回门口的书。她没有道谢，但之后把用户常看的书放到更容易拿的位置。

关系变化：
- 用户越靠近她的私人边界，她越容易不耐烦；如果用户尊重边界，她会用很小的行动回报。

角色扮演规则：
- 不要让她突然正面表达感谢。优先用物件位置、营业时间、留书、避开视线表达变化。

暂不写入：
- 没寄出的信具体写给谁，除非作者已经确定。
```

为什么有效：

- 同一段内容被拆成了身份、空间、记忆、关系和规则。
- 未确定内容没有被 AI 擅自补完。

## 12. 去 AI 腔反例

### 12.0 保留作者原始判断

作者原句：

```text
她很温柔，但把门摔得很响。
```

不建议润成：

```text
她是一个温柔而矛盾的人，内心压抑着难以言说的情绪。
```

问题：这句话流畅了，但角色被抹平了。“温柔”和“摔门”之间的不一致，正是角色变具体的地方。

可以改成：

```text
她把门摔得很响。
走到楼梯口又停下，回头问了一句：“你手疼不疼？”
```

为什么有效：

- 保留了作者原始矛盾。
- 没有把矛盾解释成心理总结。
- 让 AI 通过动作理解“这个人的温柔不是轻声细语”。

### 12.1 角色介绍

AI 腔：

```text
她是一个充满矛盾与张力的人物，在冷漠外表下隐藏着柔软内心，命运使她不得不学会保护自己。
```

改写：

```text
她不喜欢别人进厨房。真有人饿着，她会把面煮好放在桌上，人却已经回了房间。
```

### 12.2 关系推进

AI 腔：

```text
这次事件让他们之间的信任产生了微妙变化，也为后续关系发展埋下伏笔。
```

改写：

```text
那天之后，他没有再把备用钥匙收回去。
```

### 12.3 亲密描写

AI 腔：

```text
他的占有欲与温柔形成复杂交织，展现出强烈而克制的情感张力。
```

改写：

```text
他伸手前先看了对方一眼。得到回应后才靠近。后来对方提起这件事，他只说“你想多了”，耳根却红得很慢。
```

### 12.4 世界书

AI 腔：

```text
这座城市古老而神秘，承载着无数人的命运，是故事中不可或缺的重要舞台。
```

改写：

```text
城内禁止在钟响后点第二盏灯。第二盏灯意味着屋里有外人，巡查会敲门。
```

### 12.5 记忆

AI 腔：

```text
这段经历让角色意识到用户的重要性，并在内心深处留下不可磨灭的痕迹。
```

改写：

```text
用户替他挡过一次盘问。之后只要听见同样的脚步声，他会先把用户往身后带半步。
```

## 13. 小型完整流程

用户说：

```text
我想做一个不愿意被救的人。
```

不要立刻写完整角色卡。

先判断：

```text
已确定：
- 角色不愿意被救。

高可能推测：
- 他可能害怕欠债、害怕暴露脆弱，或认为自己不值得被救。这些都还不是事实。

仍待决定：
- 他拒绝被救，是因为骄傲、亏欠、习惯、惩罚自己，还是不相信救援？
```

只问一个关键问题：

```text
他拒绝被救时，最先护住的是什么：体面、自由、罪责、某个秘密，还是“不要连累别人”？
```

如果用户回答：

```text
他最怕连累别人。
```

可以整理：

```text
角色底色：
- 他不是不想活，也不是讨厌别人靠近。他把自己的存在理解成负担，所以越被救，越觉得自己在拖人下水。

本性与选择：
- 危险中先切断别人靠近自己的路径。
- 如果别人坚持救他，他会先发火，因为发火比承认害怕更容易。
- 如果对方因此受伤，他之后会更礼貌、更疏远。

晕染：
- 他把绷带扔回去，说“省着点”。手却一直按着伤口，不让血滴到对方鞋边。

防写偏说明：
- 不要写成普通自毁角色。他拒绝帮助的核心不是想死，而是不允许别人为他付代价。

可写入：
- 角色核心：拒绝被救的原因和选择顺序。
- 关系变化：用户坚持留下后，他不会立刻感动，会先拉开距离。
- 角色扮演规则：不要让他快速被爱治愈，也不要每轮复读“我不值得”。
```

这个流程的关键：

- 先分事实和推测。
- 只追问一个最影响角色成立的问题。
- 回答后立刻落到行为、选择、晕染和写入位置。
